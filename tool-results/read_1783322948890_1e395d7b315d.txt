    71→Task ID: 13
    72→Agent: main
    73→Task: Fix login not working in Preview Panel (iframe cookie blocking)
    74→
    75→Root cause:
    76→- The Preview Panel embeds the app in a cross-origin iframe. Browsers block `sameSite: 'lax'` cookies from being SET in cross-site iframes. So the login POST succeeded server-side (→ "logged in" toast) but the session cookie was never stored client-side → /api/auth/me returned user:null → user stayed on the landing page.
    77→
    78→Fix (Bearer token + localStorage, cookie kept as fallback):
    79→- src/lib/auth.ts: getSession() now reads `Authorization: Bearer <token>` header FIRST (works in any iframe/origin), then falls back to the httpOnly cookie. setSessionCookie() now also RETURNS the raw token.
    80→- src/app/api/auth/{login,signup,google}/route.ts: responses now include `token`.
    81→- src/lib/api-client.ts: added getStoredToken/setStoredToken/clearStoredToken (localStorage); apiFetch() and uploadFile() attach `Authorization: Bearer <token>` when present; 401 clears stale token.
    82→- src/hooks/use-auth.ts: login/signup/google store the returned token in localStorage before fetchMe; logout clears it; fetchMe short-circuits to null when no token (avoids 401 spam). Also resets dashboard view to 'overview' on every successful auth so users land on the Overview.
